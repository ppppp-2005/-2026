# 00-main-control SPEC

## 职责

维护整体产品蓝图、工作流规则、模块边界和验收标准。

## 当前负责内容

- 保持 `docs/blueprint` 与实际开发方向一致。
- 决定什么时候新增 workstream。
- 汇总其他线程的 `HANDOFF.md`。
- 检查是否出现重复代码、越界修改或方向偏离。
- 维护分布式 `STATE.json`、自动状态快照和本地上下文索引。
- 使用定向检索接受子线程交接，不把完整历史复制进主对话。

## 不负责内容

- 不直接实现首页 UI。
- 不直接实现项目骨架。
- 不直接接入后端。

## 当前阶段验收标准

- 目录结构已建立。
- 三个核心 workstream 文档已建立。
- 项目骨架和首页 UI 有独立线程可继续工作。
- 状态、交接、索引和线程轮换规则可由脚本校验。
