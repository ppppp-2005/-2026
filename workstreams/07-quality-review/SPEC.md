# 07-quality-review SPEC

## 职责

负责阶段性质量检查，不直接开发新功能。

## 必做内容

- 检查已完成前端页面是否符合蓝图。
- 检查 workstream 边界是否被破坏。
- 检查 mock 数据是否集中管理。
- 检查微信开发者工具是否能正常编译和切换 Tab。
- 检查 UI 是否有明显文字溢出、重叠、误导性按钮或不清楚的信息层级。

## 不做内容

- 不新增功能页面。
- 不接后端。
- 不重写已有页面。
- 不修改设计方向，除非发现明确问题。

## 验收标准

- 输出质量检查结果。
- 列出必须修复项和可选优化项。
- 总控线程根据结果决定是否进入下一个模块。
