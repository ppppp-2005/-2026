# 首页 8 个入口 UI 改版方案

## 目标

为首页功能入口区设计一版更清楚、更务实的 UI。主要服务对象是文化程度不高、手机使用经验有限的家乡工人，所以入口要做到：

- 一眼能看懂是做什么的。
- 字大、对比强、点击区域大。
- 不依赖复杂图标，不使用真实政府 logo 或单位标识。
- 后续接 API 后可以按入口跳转页面、打开外链、显示禁用或维护状态。

## 推荐方案：大字图章式 2 列入口

当前 4 列宫格每个入口较小，适合熟练用户，但对年龄偏大或文化程度较低的工人不够友好。建议改为“2 列大按钮”，每个入口包含：

- 左侧大号文字图章：用 1 个汉字表达入口。
- 右侧主标题：入口名称，字要大。
- 右侧说明：一句非常短的白话说明。
- 右上状态：可选，用于“常用”“推荐”“占位”“未开通”。

## 入口文案

| 入口 | 图章字 | 一句话说明 | 面向用户的理解 |
| --- | --- | --- | --- |
| 找工作 | 工 | 看附近岗位 | 我要找活干 |
| 企业招人 | 招 | 发布招工信息 | 老板要招工 |
| 技能培训 | 技 | 学门手艺 | 学技能、考证 |
| 看政策 | 政 | 补贴和通知 | 看补贴、看通知 |
| 校园招 | 校 | 学生就业 | 学校和学生入口 |
| 用工信息 | 信 | 企业用人 | 看哪里缺人 |
| 返乡专区 | 乡 | 回乡找机会 | 想回老家工作 |
| 培训报名 | 报 | 报名学技能 | 我要报名培训 |

文案原则：

- 标题保留现有 2 到 4 个字，不改成复杂长词。
- 说明控制在 5 到 6 个汉字左右，避免一行塞满。
- 不用“人才服务”“人力资源”等偏正式词做主文案，放到详情页或说明里更合适。

## 颜色方案

整体要像地方就业服务，不要像高端招聘 App。建议使用低饱和、强识别的颜色，每个入口固定一个色块：

| 入口 | 主色 | 背景浅色 | 用法 |
| --- | --- | --- | --- |
| 找工作 | `#1F7A4D` | `#E8F4ED` | 最高优先级，绿色代表就业服务 |
| 企业招人 | `#2F6FB2` | `#EAF1FA` | 蓝色代表企业和信息 |
| 技能培训 | `#C7792B` | `#FFF1E2` | 橙色代表学习和实操 |
| 看政策 | `#7A5A2A` | `#F7EEDB` | 棕金色代表政策文件 |
| 校园招 | `#4F6F52` | `#EEF5EA` | 稳重绿色，区分找工作 |
| 用工信息 | `#586B8F` | `#EEF2F8` | 灰蓝色代表信息 |
| 返乡专区 | `#A45F3D` | `#FFF0E8` | 暖棕色代表家乡 |
| 培训报名 | `#6B5B95` | `#F1EEFA` | 紫灰色代表报名和课程 |

避免：

- 8 个入口全是同一个绿色，用户不容易记。
- 大面积亮红色，容易像警告或错误。
- 很浅的灰字，户外看手机会不清楚。

## 视觉结构

建议入口区标题从“功能入口”改为“常用服务”，更贴近用户语言。

每个入口卡片结构：

```text
┌──────────────────────┐
│ ┌────┐  找工作    >  │
│ │ 工 │  看附近岗位    │
│ └────┘              │
└──────────────────────┘
```

设计细节：

- 卡片高度建议 `136rpx` 到 `156rpx`，点击面积足够。
- 一行 2 个入口，卡片间距 `16rpx` 到 `20rpx`。
- 图章块建议 `64rpx` 到 `72rpx`，圆角 `8rpx`，字重 700。
- 标题建议 `30rpx` 到 `32rpx`，说明建议 `22rpx` 到 `24rpx`。
- 右侧可以用简单 `>` 表示可点，不需要复杂箭头图标。
- 卡片背景用浅色，图章用深色，文字用深灰，提升可读性。

## 交互状态

后续实现时建议支持这些状态：

| 状态 | 表现 | 适用情况 |
| --- | --- | --- |
| 正常可点 | 深色图章、白底或浅色底、右侧 `>` | 已有目标页面或后续可跳转 |
| 推荐 | 右上角显示“小字标签：常用 / 推荐” | 找工作、技能培训等重点入口 |
| 未开通 | 卡片透明度降低，说明改为“暂未开通” | 后端或页面未完成 |
| 维护中 | 右上角显示“维护” | API 临时不可用 |
| 点击反馈 | 轻微变暗或按压态 | 微信小程序点击反馈 |

禁用入口不要隐藏。对低熟练用户来说，“看得到但知道暂未开通”比突然没有入口更稳定。

## 后续落地到首页的方式

确认方案后，建议在 `02-home-ui` 的下一次实现切片中这样落地：

1. 将 `miniprogram/data/home.ts` 和 `home.js` 中的 `quickEntries` 扩展为上面的字段。
2. 首页 WXML 中入口区从 4 列小图标改为 2 列大卡片。
3. WXSS 新增入口卡片、图章、说明、角标、禁用态样式。
4. 暂时只给 `找工作` 配置 `type: "tab"` 和 `/pages/jobs/index`；其他入口保持 `enabled: false` 或点击后提示“功能建设中”。
5. 等后续页面或 API 准备好，再逐个把 `route/type/enabled` 打开。

## 验收标准

- 手机宽度下一行 2 个入口，无横向滚动。
- 每个入口标题不用放大页面也能看清。
- 用户只看标题和一句说明就知道入口大概用途。
- 禁用入口有明显但不刺眼的状态提示。
- 数据结构能支持后续 API 替换和点击跳转。

## 当前结论

建议采用“2 列大字图章式入口”。这版比 4 列小宫格更占空间，但更适合本项目目标用户：入口更大、文字更清楚、意思更直白，也便于后续按入口接 API、跳页面和做状态控制。
