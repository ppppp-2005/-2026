# 13-training-signup SPEC

## 目标

实现首页“培训报名”入口对应的报名服务页，让用户查看近期班次和报名条件，并能得到明确的静态操作反馈。

## 页面范围

- 页面路由：`pages/training-signup/index`
- 页面数据：`data/training-signup.js` 与 `data/training-signup.ts`
- 包含可报名班次、名额、开班时间、地点、费用和报名条件。
- 报名按钮明确标示“报名暂未开放”，点击仅反馈“报名功能暂未开放”，不收集真实个人信息。

## 暂不实现

- 真实表单提交、身份证信息、支付、审核和后端 API。
- 不修改首页、`app.json` 或其他模块。

## 验收

- 页面能明确区分可报名、即将满员和已结束状态。
- 页面不得产生报名成功、已记录或已保存的误导性结果。
- mock 数据结构可供后续 API 和报名流程使用。
