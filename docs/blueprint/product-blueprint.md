# 家乡工人招聘小程序产品蓝图

## 当前目标

先完成微信原生小程序的完整前端框架，再开始真实后端实现。前端阶段使用 mock 数据和本地交互，必须明确区分演示状态与真实登录、投递、发布或消息发送。

## 用户与体验

- 主要用户：本地或外出找工作的工人、技能工种从业者、返乡人员。
- 次要用户：需要招工的企业或个体老板。
- 视觉：沿用绿色、米白、低圆角、大字号、短句和低理解成本。
- 交互：前端控件与状态完整可体验，但不发送请求、不持久化敏感数据、不伪造业务成功。

## 已完成并 SHIP

- 项目骨架、四个 Tab、首页与八个入口。
- 职位列表静态页。
- 企业招人、技能培训、政策、校园招聘、用工信息、返乡专区、培训班次页面。
- 静态质量复查与微信开发者工具人工验收。
- 后端架构 v1 设计；仅作为未来实现依据，当前不启动服务器代码。

## 前端框架完成线

按顺序完成以下四个切片：

1. `04-user-profile`：我的、演示登录状态、简历编辑框架、投递记录与身份视图。
2. `14-messages`：消息列表、分类/未读状态、通知与会话详情框架。
3. `03-jobs` 新切片：本地搜索筛选、职位详情、收藏与投递确认框架。
4. `05-employer` 新切片：企业资料、岗位发布/编辑、预览、候选人列表与处理框架。

随后由 `01-project-shell` 统一注册新增路由，`07-quality-review` 完成全前端编译、导航、交互、窄屏和真实性复查。全部 SHIP 后，才能启动 `06-backend-api` Phase 1。

## 技术方向

- 前端：微信原生小程序，WXML / WXSS / TypeScript / JavaScript / JSON。
- 数据：前端阶段继续使用结构化 mock，并通过 service 接口预留未来 API 适配。
- 后端：架构方案见 `docs/blueprint/backend-design-placeholder.md`，实现延期至前端框架完成。

## 当前禁止

- 真实微信登录、手机号换取、敏感资料持久化。
- 真实职位投递、企业发布、聊天发送、报名、支付或政策申报。
- 后端服务器、数据库迁移、真实 API、管理后台和生产部署。
